<p>I'm trying to write a regular expression to validate US phone number of format 
 (123)123-1234 -- true
 123-123-1234 -- true</p>
<p>every thing else in not valid.</p>
<p>I came up something like </p>
<pre><code> ^\(?([0-9]{3}\)?[-]([0-9]{3})[-]([0-9]{4})$
</code></pre>
<p>But this validates, 
123)-123-1234
(123-123-1234</p>
<p>which is NOT RIGHT.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The easiest way to match both</p>
<pre><code>^\([0-9]{3}\)[0-9]{3}-[0-9]{4}$
</code></pre>
<p>and</p>
<pre><code>^[0-9]{3}-[0-9]{3}-[0-9]{4}$
</code></pre>
<p>is to use <em>alternation</em> (<code>(...|...)</code>): specify them as two mostly-separate options:</p>
<pre><code>^(\([0-9]{3}\)|[0-9]{3}-)[0-9]{3}-[0-9]{4}$
</code></pre>
<p>By the way, when Americans put the area code in parentheses, we actually put a space after that; for example, I'd write <code>(123) 123-1234</code>, not <code>(123)123-1234</code>. So you might want to write:</p>
<pre><code>^(\([0-9]{3}\) |[0-9]{3}-)[0-9]{3}-[0-9]{4}$
</code></pre>
<p>(Though it's probably best to explicitly demonstrate the format that you expect phone numbers to be in.)</p>
